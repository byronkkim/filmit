# filmit — 오케스트레이터 지침

## 세션 시작 시 필수 루틴
1. `project_state.json` 읽기
2. 완료된 것, 진행 중인 것, 다음 할 것 파악
3. 한국어로 현재 상태 요약 후 작업 시작

---

## 서비스 개요

시청자/광고주가 원하는 영상 주제를 **Quest(퀘스트)** 로 등록하고, 크리에이터가 영상을 제작하면 에스크로 금액을 받는 영상 제작 후원 플랫폼.

- 영상 저작권은 100% 크리에이터 소유
- 결제는 후원 성격 (영상 제작 용역 계약 아님)
- 퀘스트 표시 금액 = 크리에이터 세전 실수령액

---

## 사용자 3종

| 유저 | 역할 | 수수료 |
|------|------|--------|
| 시청자 | 소액 다수 → AI가 묶어 Quest 생성 | 크리에이터 보상 + 10% |
| 광고주 | 단독 고액, 브랜드 캠페인 | 크리에이터 보상 + 15% |
| 크리에이터 | 퀘스트 수행, 영상 제작, 보상 수령 | 세전 수령 (원천징수 3.3% 차감) |

## 핵심 개념

### Quest (퀘스트)
- AI가 유사 요청들을 임베딩(pgvector) 기반으로 묶어 자동 생성
- 시청자가 새 요청 입력 → 유사 퀘스트 탐색 먼저 → 없으면 새 퀘스트 등록
- 필수 조건(Main Quest) + 개별 요건(Sub-quest) 구조

### Sub-quest (서브퀘스트)
- 각 시청자의 세부 요건이 서브퀘스트 체크리스트로 등록
- 크리에이터 달성 시 해당 후원자 풀 지급, 미달성 시 환불
- 광고주는 서브퀘스트별 금액 개별 설정 가능 (시청자는 통합)

### 수수료 구조 (핵심)
```
크리에이터 보상금액 (퀘스트 표시, 세전)
+ 플랫폼 수수료 (시청자 10% / 광고주 15%)
+ PG 수수료 (~3.3%)
= 시청자/광고주 실결제액
```
- 에스크로 보관액 = 크리에이터 보상금액만
- 플랫폼 수수료는 결제 즉시 플랫폼 계좌 분리 입금

### 정산 규칙
- 필수 조건 미달 → 전액 환불 (크리에이터 수령 0)
- 필수 달성 + 서브퀘스트 부분 달성 → 달성분 지급 + 미달성분 환불
- 영상 완성 후 7일 이의신청 기간 → 이후 자동 정산
- 시청자 '맞아요' 클릭 시 즉시 정산 트리거

### 환불 정책
- 크리에이터 귀책(미완성, 30일 초과) → 플랫폼 수수료 포함 전액 환불
- 크리에이터 수락 전 시청자 취소 → 전액 즉시 환불
- 크리에이터 수락 후 → 환불 불가

## 광고주 전용 기능

- 서브퀘스트별 금액 개별 설정
- 크리에이터 노출 조건 필터: 구독자 수, 카테고리, 특정 채널명
- 경쟁 방식: 여러 크리에이터 동시 도전 → 광고주 선택 → 선택된 크리에이터만 정산
- 영상 소유권 이전 요청 가능 (수수료 +5%, 크리에이터 동의 필요)
- 비공개 퀘스트 (조건 맞는 크리에이터에게만 노출)
- 결제 즉시 전자세금계산서 자동 발행

---

## 크리에이터 전용 기능

- 채널 연동: YouTube Data API v3로 구독자수·카테고리 자동 분석
- 등급 시스템: 완료 퀘스트수·달성률·이의신청 비율 기반 자동 산정
  - 등급 높을수록 퀘스트 피드 상위 노출, 광고주 퀘스트 우선 초대
- KYC: 가입 시 주민번호 뒷자리(AES-256 암호화) 또는 사업자번호 수집
- 정산 계좌: 토스페이먼츠 1원 인증

## 세금 처리 (자동화 필수)

| 항목 | 내용 |
|------|------|
| 원천징수 | 정산 시 3.3% 자동 차감 |
| 원천징수영수증 | 정산 즉시 PDF 자동 발행, 이메일 발송 |
| 지급명세서 | 매년 2월 말 국세청 홈택스 자동 제출 |
| 원천세 납부 | 매월 10일 Cron Job 자동 집계·신고 |
| 연간 CSV | 크리에이터 대시보드에서 다운로드 (5월 종합소득세 신고용) |
| 에스크로 계좌 | 플랫폼 운영 계좌와 반드시 분리 (예수금 처리) |
| 세금계산서 | 광고주 결제 즉시 자동 발행 (스마트빌 API) |

## 영상 검증 파이프라인 (자동화)

```
업로드 → FFmpeg(품질체크) → Audible Magic(저작권) 
→ AssemblyAI LeMUR(STT+요약) → Claude API(주제매칭) 
→ Google Video Intelligence(유해콘텐츠) → pHash(중복탐지) 
→ 통과/검토/거부
```

- 통과(90%+): 자동 공개 + 7일 이의신청 시작
- 검토(50~90%): 관리자 큐
- 거부: 자동 환불 + 크리에이터 사유 안내

---

## AI 자동화 항목

- 퀘스트 생성: 시청자 입력 → Claude API로 핵심/부가 분해 → Quest + Sub-quest 구조화
- 유사 요청 탐지: pgvector 코사인 유사도 ≥ 0.80 → 동일 퀘스트로 묶음
- CS 자동응대: 이용약관+FAQ RAG → 90%+ 자동 해소
- 부적절 요청 필터: 입력 즉시 Claude API 실시간 검사 (500ms 디바운스)
- 분쟁 판정: 주제 불일치 신고 → Claude API가 요청문 vs 영상요약 비교 → 60%↑ 자동환불

## DB 핵심 테이블

```
users (user_type: viewer / advertiser / creator)
quests (main quest, embedding 컬럼)
sub_quests (quest_id FK, amount, achiever_id)
pledges (quest_id, sub_quest_id FK, user_id, amount, platform_fee)
creators (channel_url, subscriber_count, categories, grade)
videos (quest_id, creator_id, status, ai_verification_result)
settlements (pledge_id, amount, withholding_tax, net_amount, status)
```
