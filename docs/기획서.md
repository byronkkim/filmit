## 1. 서비스 개요

### 한 줄 설명
시청자/광고주가 원하는 영상 주제를 **Quest(퀘스트)** 로 등록하면, 크리에이터가 YouTube 영상으로 제작하고 에스크로 금액을 받는 **영상 제작 후원 플랫폼**.

### 핵심 가치
- **영상 저작권은 100% 크리에이터 소유**
- 결제는 **후원 성격** (영상 제작 용역 계약 아님)
- 퀘스트 표시 금액 = 크리에이터 **세전 실수령액**
- **소액 다수 모금**: 천원짜리 후원이 모여 10만원 퀘스트 완성
- 플랫폼은 **철저히 심판 역할** (판정에 개입하지 않음)

### 감성/톤
- 활기차고 빠른 템포
- 게임처럼 퀘스트를 완료하고 별(⭐)을 모으는 느낌
- 어뷰징은 엄격하게 차단, 선한 유저는 편하게

## 3. 핵심 개념

### 3-1. Quest (퀘스트)

시청자/광고주가 만들어주길 원하는 영상 요청.

- **AI가 유사 요청들을 임베딩(pgvector) 기반으로 묶어 자동 생성**
- 시청자 입력 → 유사 퀘스트 탐색 먼저 → 없으면 새 퀘스트 등록
- 구조: **메인 조건(필수)** + **서브 조건(보너스)**

### 3-2. Sub-quest (서브퀘스트) — ⭐ 별 시스템

- 서브퀘스트는 **금액과 무관**, **명예(별)** 시스템
- 메인 퀘스트 달성 → 전액 정산, 서브퀘스트는 ⭐ 보너스
- 후원자 투표로 달성 여부 결정 (리뷰 역할)
- 별은 크리에이터 프로필에 누적 → 등급에 반영

### 3-3. 퀘스트 타입 3종 (예정)

| 타입 | 설명 | 적합 상황 |
|------|------|------|
| 🎯 **단독 퀘스트** | 크리에이터 1명이 수락하면 즉시 잠금 | 소액, 일반 요청 |
| 🏆 **경쟁 퀘스트** | 여러 크리에이터 기획서 제출 → 후원자 투표로 1명 선정 | 고액, 퀄리티 중요 |
| 💌 **지명 퀘스트** | 특정 크리에이터 지정, 수락권은 크리에이터에게 | 팬덤 기반 |

> 초기 MVP는 **단독 + 지명**만 구현. 경쟁은 후속.

### 3-4. filmit 크레딧 (영상 내 출처 표기)

후원자 만족도 + 플랫폼 홍보를 위한 장치:

- 영상 설명란: "이 영상은 filmit 후원자 N명의 요청으로 제작되었습니다 🎬"
- 영상 내 한마디: "오늘 영상은 filmit에서 요청받아서 만들었어요"
- 고정 댓글: filmit 퀘스트 링크

**강제 아닌 인센티브**: 기본 서브퀘스트로 자동 추가 → 달성 시 ⭐ 보너스

## 4. 결제 및 정산

### 4-1. 수수료 구조

```
크리에이터 보상금액 (퀘스트 표시, 세전)
+ 플랫폼 수수료 (시청자 10% / 광고주 15%)
+ PG 수수료 (~3.3%)
= 시청자/광고주 실결제액
```

- **에스크로 보관액 = 크리에이터 보상금액만**
- 플랫폼 수수료는 결제 즉시 플랫폼 계좌 분리 입금
- 모든 금액 계산은 **정수(원 단위)**, 부동소수점 금지

### 4-2. 정산 규칙

- 메인 퀘스트 달성 → **전액 정산**
- 메인 퀘스트 미달 → **전액 환불** (크리에이터 수령 0)
- 영상 완성 후 투표 기간 종료 → 자동 정산
- 서브퀘스트는 정산에 영향 없음 (⭐만 부여)
- 정산 시 **원천징수 3.3% 자동 차감** → 실수령액 = 보상금 × 96.7%

### 4-3. 환불 정책

| 상황 | 환불 내역 |
|------|---------|
| 크리에이터 수락 전 시청자 취소 | 전액 즉시 환불 |
| 크리에이터 수락 후 | 환불 불가 |
| 크리에이터 귀책 (미완성, 데드라인 초과) | 플랫폼 수수료 포함 전액 환불 |
| 판정 거부 확정 | 후원자에게 전액 환불 (플랫폼 수수료 포함) |

## 5. 판정 로직

### 5-1. 기본 원칙

- **AI + 후원자 투표 동시 진행**
- 무응답 = **승인으로 간주** (적극적 거부가 다수여야 거부)
- 플랫폼은 개입하지 않음 (규칙대로만)

### 5-2. 판정 매트릭스

| AI 판정 | 후원자 판정 | 결과 |
|---------|-----------|------|
| ✅ OK | ≥ 50% OK | ✅ **즉시 정산** |
| ❌ No | ≥ 67% OK | ✅ **즉시 정산** |
| ✅ OK | < 50% OK | ❌ 환불 |
| ❌ No | < 67% OK | ❌ 환불 |

### 5-3. 후원자 투표 규칙

- 분모 = **해당 퀘스트 전체 후원자 수** (응답 여부 무관)
- 기간 내 무응답 → **승인으로 간주**
- 투표권: 해당 퀘스트에 **후원 기록이 있는 사용자만**

### 5-4. 투표 기간 (총 후원액별 차등)

| 총 후원액 | 투표 기간 | 근거 |
|-----------|--------|------|
| ~10만원 미만 | **24시간** | 소액, 빠른 템포 / 즉각적 보상감 |
| 10만~100만원 | **3일** | 중간 규모, 적정한 숙고 |
| 100만원 이상 | **7일** | 고액, 충분한 검토 |

### 5-5. AI 주제 매칭 방식

```
크리에이터가 YouTube URL 제출
  ↓
YouTube API로 메타데이터 + 자막 가져오기 (무료)
  ↓
자막 기반 Claude API 주제 매칭
  ↓
판정 점수 산출 (OK/No)
```

- **비용 절감**: YouTube 자막 우선, 없을 때만 AssemblyAI STT
- 자막 없는 영상(10분): ~$0.08~0.11 (약 110~150원)
- 시각적 조건(예: "검은 모자 쓰기")은 AI 검증 어려움 → 후원자 투표에 위임

## 6. 어뷰징 방지

### 6-1. 크리에이터 측 어뷰징 대응

| 어뷰징 | 방어 로직 |
|------|---------|
| **수락 후 잠수** | 14일 deadline → Cron 자동 취소 → 3회 누적 시 30일 제재 |
| **예전 영상 재탕** | `accepted_at` 이후 업로드된 영상만 허용 (YouTube publishedAt 검증) |
| **같은 영상 여러 퀘스트** | `youtube_video_id` UNIQUE 제약 + 명시적 체크 |
| **남의 채널 영상 도용** | 등록된 `channel_id`와 영상 채널 일치 검증 |
| **저품질 영상 난사** | AI + 후원자 검증 → 거부 시 신뢰 점수 감점 |

### 6-2. 후원자 측 어뷰징 대응 (예정)

| 어뷰징 | 방어 |
|------|------|
| 악의적 거부 투표 | 무응답=승인, 2/3 다수결 |
| 가짜 계정 동원 | 후원 기록 있는 사용자만 투표권 |
| 반복 환불 신청 | 신뢰도 시스템 (가중치 감소) |

### 6-3. 패널티 시스템

**크리에이터 제재 (현재 구현됨)**:
- 데드라인 초과: `abandoned_quests` +1
- 누적 3회: `suspended_until` = 30일 후 → 새 퀘스트 수락 불가

**추가 예정**:
- 신뢰 점수 기반 등급 자동 조정
- 영상 거부 시 감점
- 후원자 투표 가중치

## 7. KYC 본인인증

### 7-1. 기본 정책

- **정산을 위해 필수**
- AES-256-GCM 암호화 저장
- 평문은 서버 메모리에서 즉시 폐기
- 복호화는 **정산 시에만** 수행

### 7-2. 인증 유형

| 유형 | 수집 항목 | 용도 |
|------|--------|------|
| **개인** | 주민번호 뒷자리 7자리 | 원천징수 신고 |
| **사업자** | 사업자등록번호 10자리 | 세금계산서 발행 |

### 7-3. DB 저장 구조

```
creators.kyc_type        // 'individual' | 'business'
creators.kyc_encrypted   // AES-256-GCM 암호문 (hex)
creators.kyc_iv          // 암호화 IV (hex)
creators.kyc_verified    // 인증 완료 여부
creators.kyc_verified_at // 인증 완료 시각
```

## 8. 크리에이터 시스템

### 8-1. 채널 연동

- YouTube Data API v3로 채널 자동 분석
- 입력: 채널 URL (`@handle`, `/channel/UC...`, `/c/...`, `/user/...` 형식 지원)
- 자동 수집: 채널명, 구독자수, 카테고리(topicDetails), 썸네일

### 8-2. 정산 계좌

- 은행 선택 + 계좌번호 + 예금주
- 지원 은행 17개 (국민, 신한, 우리, 하나, 농협, SC, 씨티, 우체국, 지방은행, 인터넷은행 등)
- 1원 인증 (토스페이먼츠 연동 예정)

### 8-3. 등급 시스템

| 등급 | 조건 | 혜택 |
|------|------|------|
| 플래티넘 | 신뢰 점수 90+ | 고액 퀘스트 우선 초대 |
| 골드 | 70+ | 퀘스트 상단 노출 |
| 실버 | 50+ | 기본 |
| 브론즈 | 0~49 | 동시 수락 제한 (예정) |

### 8-4. 크리에이터 온보딩

3-step 플로우:
1. YouTube 채널 URL 입력 → 자동 조회
2. 정산 계좌 (선택 가능, 나중에 등록 가능)
3. KYC 본인인증 (선택 가능, 나중에 등록 가능)

## 10. DB 스키마

### 10-1. 주요 테이블

```
users             - 유저 기본 정보 (user_type: viewer/advertiser/creator)
creators          - 크리에이터 프로필 (채널, 등급, KYC, 정산계좌, 제재)
quests            - 퀘스트 메인 (embedding 컬럼 포함)
sub_quests        - 서브퀘스트 (별 시스템, amount 없음)
sub_quest_votes   - 서브퀘스트 달성 투표 (후원자 리뷰)
pledges           - 후원/결제 기록
videos            - 제출된 영상 (YouTube 메타데이터 포함)
settlements       - 정산 (원천징수 포함)
disputes          - 이의신청
```

### 10-2. 주요 enum

```
user_type:          viewer / advertiser / creator
quest_status:       draft / open / in_progress / reviewing / completed / cancelled
sub_quest_status:   pending / achieved / failed
pledge_status:      pending / escrowed / released / refunded / partial_refund
video_status:       uploaded / verifying / approved / manual_review / rejected / published
settlement_status:  pending / processing / completed / failed
creator_grade:      bronze / silver / gold / platinum
dispute_status:     open / auto_resolved / admin_review / closed
```

### 10-3. 마이그레이션 이력

| 파일 | 내용 |
|------|------|
| 00001 | 초기 스키마 (9개 테이블 + pgvector + RLS + 트리거) |
| 00002 | RLS 정책 |
| 00003 | `increment_quest_pledged_amount` 함수 |
| 00004 | KYC 컬럼 (`kyc_type`, `kyc_encrypted`, `kyc_iv`, `kyc_verified`, `kyc_verified_at`) |
| 00005 | 서브퀘스트 별 시스템 (`amount` 제거, `star_votes_yes/no` 추가, `sub_quest_votes` 테이블) |
| 00006 | 영상 무결성 (UNIQUE 제약, `youtube_published_at`, `abandoned_quests`, `suspended_until`) |

## 부록: 개발 규칙

- 한국어로 소통
- 세션 시작 시 반드시 `project_state.json` 읽기
- 기능 구현 후 `project_state.json` 업데이트
- 환경변수는 `.env.local`에만 저장, **절대 커밋 금지**
- 모든 금액 계산은 **정수(원 단위)**, 부동소수점 사용 금지
- Supabase **RLS 정책 모든 테이블에 필수 적용**
- 에스크로 계좌는 별도 계좌로 분리 (예수금 처리)
- 커밋 메시지 한국어 OK, Co-Authored-By 태그 포함
